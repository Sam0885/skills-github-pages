⚡
SKILL.md – Struktur & Aufbau
Dateiformat, Frontmatter, Body und Ressourcen
🛠️ Technical 💻 Developer 📄 SKILL.md
⏱ ~10 Min.
0%
✓ ▼
Skills werden als Ordner mit einer SKILL.md-Datei angelegt. Der Ordnername muss identisch mit dem name-Feld im YAML-Frontmatter sein – sonst lädt der Skill lautlos nicht. 
---
name: gorelate-html-report         # Pflicht: kebab-case, max 64 Zeichen
description: >-                      # Pflicht: WANN + WAS, max 1024 Zeichen
  Erstellt HTML-Dokumente im gorelate Corporate Design.
  Nutzen wenn: HTML report, gorelate design, Firmenbericht.
argument-hint: 'Thema des Dokuments'  # Optional: Hinweis im Chat-Input
user-invocable: true               # Standard: als /slash-command nutzbar
context: inline                    # 'inline' oder 'fork' (Subagent)
---

Copilot Studio Skills
Pro-Code Erweiterungen via Bot Framework / Agents SDK
🏗️ AI Architect 💻 Developer 🤖 Copilot Studio
⏱ ~10 Min.
0%
✓ ▼
In Copilot Studio ist ein „Skill" ein pro-code Bot (Microsoft 365 Agents SDK), der als Erweiterung in einen Copilot Studio Agent eingebettet wird. Ganz anderes Konzept als GitHub Copilot Skills!
⚠️ Terminologie-Falle: "Skills" in Copilot Studio = Bot Framework Bots. "Skills" in GitHub Copilot = SKILL.md Dateien. Beide Konzepte sind grundlegend verschieden!
Tool-TypKomplexitätWann nutzen
Connector🟢 NiedrigStandard API Calls (Office 365, Dataverse …)
Agent Flow🟢 Niedrig–MittelPower Automate Logik mit AI-Steps
REST API Tool🟡 MittelCustom REST API direkt anbinden
MCP Server🟡 Mittel–HochKomplexe externe Tool-Integrationen
Skill🔴 HochBestehender Bot Framework Bot einbinden

✓
Voraussetzungen: Agents SDK, Azure Deployment
✓

✓
Skill im Topic via "Add a tool" einbinden
✓
Inputs/Outputs mit Topic-Variablen verknüpfen
✓
Tool-Limit: max. 128, empfohlen max. 25–30 pro Agent
